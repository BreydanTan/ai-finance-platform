# 基础操作提示词库 📝

本文档包含日常开发中常用的 AI 协作提示词。

---

## 环境搭建相关

### 1. 安装依赖失败

```
我在运行 npm install 时遇到以下错误：
[粘贴错误信息]

我的环境：
- 操作系统: [Windows/macOS/Linux]
- Node.js 版本: [运行 node -v 的结果]
- npm 版本: [运行 npm -v 的结果]

请帮我分析问题并提供解决方案。
```

### 2. 启动开发服务器失败

```
我运行 npm run dev 时出现错误：
[粘贴错误信息]

项目是从 GitHub 克隆的，我已经运行了 npm install。
请告诉我可能的原因和解决步骤。
```

### 3. 配置环境变量

```
我需要配置项目的环境变量。

项目使用以下服务：
- 数据库: PostgreSQL (Supabase)
- 认证: Clerk
- AI: Google Gemini

请告诉我：
1. 需要哪些环境变量
2. 如何获取这些密钥
3. .env 文件的格式示例
```

## 使用技巧

### 提示词优化建议

1. **提供上下文**
   - 说明项目背景
   - 使用的技术栈
   - 当前遇到的问题

2. **粘贴代码**
   - 包含相关的完整代码
   - 标注文件路径
   - 说明代码作用

3. **明确需求**
   - 具体说明想要什么结果
   - 列出约束条件
   - 说明优先级

4. **提供错误信息**
   - 完整的错误堆栈
   - 浏览器控制台输出
   - 终端错误信息

### 示例：好的提示词 vs 不好的提示词

❌ **不好的提示词**:
```
我的代码不工作，帮我修一下。
```

✅ **好的提示词**:
```
我在实现用户登录功能时遇到问题。

环境：
- Next.js 15
- Clerk 认证
- 本地开发环境

代码（actions/auth.js）：
[粘贴代码]

错误信息：
[粘贴错误]

预期行为：用户登录后跳转到 /dashboard
实际行为：页面刷新但停留在登录页

请帮我找出问题并提供解决方案。
```
